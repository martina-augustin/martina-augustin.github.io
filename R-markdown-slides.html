Creating Slides Program­mati­cally with R Markdown

Traditional presentation tools like PowerPoint dominate the scene, but there's a different, more dynamic way to create slides: using R and R Markdown. While this approach might seem intimidating, requiring some knowledge of R, HTML, and CSS, the results are often worth the effort. Even with limited coding knowledge, you can create unique, engaging, and visually appealing presentations. Plus, updating data plots becomes a breeze—just reload the webpage instead of copy-pasting new plots into PowerPoint slides.

Example 1 - Presentation about R

In my company, a group of R enthusiasts, including myself, decided to present to our team about using R for data analysis. We believed it would be fitting to showcase R's capabilities by using R Markdown for the presentation.

Goal

Our objective was to demonstrate how R could be utilized for the entire process, from data processing to creating the final presentation slides.

Result

We chose reveal.js for our slides due to its polished appearance and robust features. Reveal.js is a JavaScript library that facilitates the creation of slides and is available in the R package revealjs. Here are some features we utilized:

  • Transition Options: Different transition effects between slides.
  • Directional Slides: Ability to change slides (and sub-slides) in four directions, indicated by arrows in the bottom right corner.
  • Zoom: Zoom in on any slide by pressing the 'Alt' key and left-clicking.
  • Blackboard Mode: Switch to a blackboard or whiteboard by pressing the 'B' key.
  • Pause Function: Dim the slides to shift focus to the speaker by pressing the '.' key.
  • Overview Mode: Press the 'Esc' key to view an overview of the slides and jump between them.
These features can be explored in the examples below. Navigate through the slides by clicking or using the keyboard arrows in all directions, as indicated by the options in the bottom right corner of the slides.

Example 2 - Quick report for Toastmasters meeting

In this second example, I used R Markdown with reveal.js to create an 'Ah counter' report for a Toastmasters meeting. The role of the Ah counter is to track and report the use of filler words (such as "ah," "um," "er") during the meeting.

Goal

This project aimed to present the Ah counter's report in a visually engaging way, giving participants insight into their use of filler words. The challenge was to collect, process, and present the data within the meeting's duration.

Result

As participants spoke, I entered their filler word counts into a prepared spreadsheet. Towards the end of the meeting, shortly before the Ah counter report, I ran the code to generate the slides. The code took the CSV spreadsheet, transformed the data, constructed plots, and inserted words and counts into predefined positions. For example, one slide showed the top five filler words used, while others highlighted participants who spoke without using any filler words.
