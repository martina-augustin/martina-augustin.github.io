R Markdown. Creating slides program­mati­cally.

The following two examples show presentation slides made in a different fashion than the usual PowerPoint and the likes. Creating such slides may seem daunting as it requires knowledge of R and R Markdown, and ideally some html and css for better customization. The time to code this as compared to using regular 'click' software may also be demotivating.
However I believe the results are worth the extra effort and, even with a limited knowledge of R, html or css, one can create a unique, engaging and appealing presentation. Perhaps just as importantly, should new data be available, data plots in the presentation based on coding scripts can be updated by simply reloading the webpage, much to the image of a one-stop-shop for all reporting tasks in contrast to the hassle of copy/paste new plots into Powerpoint slides.

Example 1 - Work presentation about R

In my company, a few R enthusiasts and I presented to the team about using R for data analysis. We agreed that it would be a shame to speak about R and not directly show one of its numerous features: R Markdown for presenting the outputs.

Goal

The goal was to show how R could be used for the entire analysis starting from data processing to the final slides.

Result

Whilst there are multiple options to create slides with R Markdown, I used reveal.js for its appearance and options. Reveal.js is a JavaScript library to create slides and it is also implemented in R package revealjs. Some of its features applicable within R are:

  • different transition options between the slides,
  • changing slides (and sub-slides) in four possible directions; this is shown by the arrows on the bottom right corner,
  • zoom in on any slide by pressing 'Alt' key and left click,
  • switching to a blackboard (or a whiteboard) by pressing 'B' key,
  • 'pause' the slides to shift audiences focus from the text and visuals to the speaker by pressing '.'
  • and 'Esc' key to give an overview of the slides and jump between slides.
All of these features can be tested in the examples below.

N.B.: browse the slides by clicking or/and using the keyboard arrows in all directions, as prompted by the options shown in bottom right corner of the slides.

Example 2 - Quick report for Toastmasters meeting

In this second example I used R Markdown with reveal.js to make an 'Ah counter' report. The role of the Ah counter in Toastmasters is to listen carefully throughout the meeting and record the number of 'filler words' (ah, em, err) for everyone speaking in the meeting. At the end the Ah counter gives a (generally verbal) report about everyone's use of filler words. These reports help participants to notice which words they overuse and help them track their progress over time.

Goal

Preparing the report in R was a fun project to present the results a little differently and to give participants a visual hint of their filler words use. The challenge was also to make sure the data can be collected, processed and presented in a very limited time (within the duration of the meeting).

Result

The filler word counts were inserted into a prepared spreadsheet during the meeting as the participants spoke. Towards the end of the meeting shortly before the time for the Ah counter report, the code was run to generate the slides. The code takes the csv spreadsheet, transforms the data, constructs the plots and inserts words and counts in pre-set places. One example is the slide showing the top five filler words used in the meeting, or the slides generated for each participant who spoke without saying any filler word. The code to make the slides can be found here.

N.B.: browse the slides by clicking or/and using the keyboard arrows in all directions, as prompted by the options shown in bottom right corner of the slides.
