Why R ?

  • it’s great for recurring tasks and automation
  • keeps processes documented and reproducible
  • it can cover the entire analytics process from data cleaning to creating a final report
  • it can be used together with other data languages
  • it can create beautiful visualisations
  • it can be used within other tools

…and what about Python…?




Python is an open-source programming language similar to R in many functionalities especially since the development of numpy and pandas.

  • developed by Guido van Rossum in 1980s

  • an important goal of Python’s developers is keeping it fun to use

  • simpler, less-cluttered syntax and grammar while giving developers a choice in their coding methodology

Differences


  • computer scientists
  • machine learning
  • productionising analysis
  • app development

  • statistitians
  • similarities with SQL (dplyr package)
  • producing reports

Python was developed by computer scientists and originally was not meant for data analytics

  • better for machine learning than R

  • better for productionising analysis

  • wider use including app development

R was primarily developed by statistitians and for statistics and data analytics

  • easier to do quick analysis

  • advantage when it comes to producing reports thanks to R Markdown

Similarities

  • open-source
  • extensible with modules (packages)
  • easy to learn (for non programmers)
  • used for automation
  • used for data analysis
  • supporting community
  • used in many industries
